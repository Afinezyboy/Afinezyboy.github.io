gitignore

开始前，建议先保存一个.gitignore文件，用来声明哪些文件不用于版本控制。

关于gitignore的具体用法，见 CSDN博客：gitignore 语法 。

万能模板1：保留任意深度目录下的某类文件

# 忽略所有文件类型
# ignore all dictionalry files
/**/*

# 除了所有的目录层级
# except the dictionay root (for the following rules)
!/**/

# 包含所有层级下的以下文件类型
# except the following suffixes in any dictionaries
!/**/*.py
!/**/*.pscx
!/**/*.pswx
!/**/*.pslx
!/**/*.md

# 包含.gitignore文件
# except the gitignore file
!.gitignore

git命令

基本操作和设置

  • git version：查询当前git版本
  • git --global ：设置用户名和密码
    • git config --global user.name "my-name"
    • git config --global user.email "my-email-box@site"
  • git add：添加文件用于版本控制
    • git add --all：添加所有的文件
  • git status：查看当前git状态
  • git commit -m 'some messages'：提交到当前branch

远程仓库

  • git clone：从远程仓库下载下来
  • git remote rm origin：删除主机origin的url链接
  • git remote add origin [url: https://github.???.git]：将[url]指定的远程git仓库设置为origin
  • git push origin [branch_name]：将更改推送到远程仓库(origin)的[branch_name]分支。

版本回溯

  • git log：提交历史（指针指向head）
    • git log --oneline：仅包含一行的，更简洁的提交信息
  • git reflog：操作记录，用于追溯之前的提交历史很有用
  • git reset：版本回退
    • git reset HEAD^：回退所有内容到上一个版本
    • git reset HEAD^ hello.php： 回退 hello.php 文件的版本到上一个版本
    • git reset 052e：回退到指定版本

分支管理

  • git branch：查看所有分支
  • git branch [new_branch]：创建新分支[new_branch]
  • git checkout [to_branch]：跳转到[to_branch]
  • git merge [branch_2]：将当前分支与[branch_2]合并
  • git branch -d [branch]：删除分支[branch]

清理缓存

  • git rm -r --cached .：清理缓存，回到git add之前的状态

问题记录

vscode atal: unable to access Recv failure: Connection was reset

解决方法：输入以下命令

git config --global --unset http.proxy
git config --global --unset https.proxy
