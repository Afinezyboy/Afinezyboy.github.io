尝个鲜

搭建Home主页

搭建Home主页至少需要两个文件，一个文件为.\layouts\index.html，负责主页的布局，实际上就是主页内容的模板；另一个文件为.\content\_index.md，作为主页的主内容。

下面的代码展示.\layouts\index.html内的基本代码，是一个最基本的主页配置。

<!DOCTYPE html>
<html lang="en-US">
    <head>
        <meta charset="UTF-8">
        <title>{{.Site.Title}}</title>
    </head>
    <body>  
        <h1>{{.Site.Title}}</h1> 
        {{.Content}}
    </body>
</html>

下面解释这里的一些关键部分，即双括号{{...}}内的东西代表着什么。

  • .Site.Title表示从congif.toml内定义的Title属性。
  • .Content表示从.\Content\目录下读取的主页文件，Hugo会读取这个目录下的_index.md文件

搭建Content内容页

与主页相同，搭建Contern内容页也需要两个文件：.\layout\_default\single.html，作为网页的模板。一个较为简单的模板如下（只给出<body>...</body>部分）：

<body>  
    <h1>{{.Site.Title}}</h1> 
    <h2>{{.Title}}</h2> 
    {{.Content}}
</body>

另一个文件是包含内容的Markdown文件，存放在.\content\中。作为内容，需要包含一些额外的信息，放在Markdown文件的最上方，称为front matter，通常由toml或yaml标记语言写成，或者也可以使用json。但我们不必自己写这段front matter，而是可以基于模板生成。模板文件位于.\archetypes\default.md，只需要通过命令:

> hugo new about.md

就可以从模板文件直接生成一个包含默认front matter的about.md文件。打开后发现已经有一些内容，这就是hugo识别文件所需要的front matter，是用ymal写成的：

---
title: "about"
date: 2022-03-12T22:07:54+08:00
draft: true
---

在front matter后面尽情开始创作吧！写完之后，将draft的状态更改为false，就能够在下一次的发布中见到它！
