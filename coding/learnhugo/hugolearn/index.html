Chapter 3 使用模板

2020-10-16
3 min read

使用模板

现在我们开始介绍如何通过使用模板，让网页搭建更加容易。一个最小化的Hugo模板包含以下文件（结构）：

layouts
    ├─index.html
    ├─_default
        ├─list.html
        ├─single.html

其中index.html是主页的模板，list.html是文章列表页的模板，而single.html是文章内容页的模板。理解了这些，就好理解后面的内容了。

创建自己的模板

我们通过这行命令构建自己的模板：

hugo new theme [theme-name]

比如我们通过hugo new theme mytheme创建了一个名为mytheme的主题，其包含以下文件（结构）：

mytheme
    │  LICENSE
    │  theme.toml
    ├──archetypes
    │      default.md
    ├──layouts
    │  │  404.html
    │  │  index.html
    │  ├──partials
    │  │      footer.html
    │  │      head.html
    │  │      header.html
    │  └──_default
    │          baseof.html
    │          list.html
    │          single.html
    └──static
        ├─css
        └─js

我们发现这个目录下也包括archetypes和layouts文件夹。另外还有一些之前没有接触过的文件和文件夹，他们的作用是：

  • theme.toml：模板的配置文件，存放一些配置参数；
  • partials：用来存放网页的head、页眉header、页脚footer等模板；
  • static：用来配置网页的样式ccs和动作js。

我们可以把之前创建的layouts文件夹复制替换过来，在根目录下的config.toml加上一句：

theme = 'mytheme'

再次运行hugo server -w，又能够看到我们的网页了。

未来，我们所做的一切都在模板里进行。

编辑内容、页眉和页脚

在前面的例子中，我们依然需要动手编辑index.html和_default/single.html中的元素布局。Hugo为我们提供了一个更底层的模板baseof.html，打开它后我们发现已经写了一些内容：

<!DOCTYPE html>
<html>
    {{- partial "head.html" . -}}
    <body>
        {{- partial "header.html" . -}}
        <div id="content">
        {{- block "main" . }}{{- end }}
        </div>
        {{- partial "footer.html" . -}}
    </body>
</html>

这里给我们提供了编辑head，header和footer的地方。他们的相应位置都在partial文件夹下。

<head>

<head>元素是所有头部元素的容器。元素必须包含文档的标题<title>,可以包含脚本<script>、样式<style>、<meta>信息以及其他更多的标签。我们在head.html内写入以下内容：

<head>
    <meta charset="UTF-8">
    <meta name="viewpoint", content="width=device-width, intial-scale=1">
    <title>{{.Site.Title}}</title>
</head>

第一行<meta>声明了字符编码，第二行meta则允许浏览器按照自身宽度调整页面宽度，第三行title表示网站名。

<header>

<header>是html的页眉，我们在这里放入网站标题。另外，我们把导航标签<nav>也加入header.html文件中，其中<nav>标签是HTML5中的新标签。整个header.html文件内容如下：

<header>
    <h1>{{.Site.Title}}</h1>
</header>

<nav>
    <a href="/">Home</a>
    <a href="/hugolearn">Learn Hugo</a>
    <a href="/about">About</a>
</nav>

footer

最后，我们在页脚文件footer.html内写入以下信息：

<footer>
    <small>Copyright {{now.Format "2022"}} Me.</small>
</footer>

模板替换

index.html内：

{{ define "main" }}

    {{.Content}}

{{ end }}

_default/single.html内：

{{ define "main" }}

    <h2>{{.Title}}</h2>
    {{.Content}}

{{ end }}
